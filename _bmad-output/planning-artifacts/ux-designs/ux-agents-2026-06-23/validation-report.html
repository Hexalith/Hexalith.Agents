The extra reviewer lenses sharpen the picture: accessibility is directionally strong, but governance and implementation readiness are not safe to hand off as final. Treat the current pair as a strong draft that needs blocking amendments before architecture and story-dev rely on it.

Governance review

review-governance.md

Strong draft, but not sufficient as a downstream implementation contract for every security-sensitive path.

critical

Approver authorization does not require current Source Conversation access

EXPERIENCE.md:89
Policy authorization could leak generated content to a same-tenant Party without current Source Conversation access.
Fix: Require both Approver Policy authorization and fresh Source Conversation access for proposal discovery/detail/action paths.
critical

Conversation invocation is unresolved

EXPERIENCE.md:52
The primary runtime entry point leaves identity attribution, prompt capture, denial copy, and proposed-vs-posted rendering open.
Fix: Make OQ-1 a blocking UX governance gate before invocation stories.
high

Action-time reauthorization is missing

EXPERIENCE.md:123
The model lacks stale/revalidation states when permissions or dependencies change while an editor is open.
Fix: Add revalidating, authorization stale, and denied after revalidation states.
high

Proposal discovery and notification are not privacy-bounded

EXPERIENCE.md:91
Queue and notification payload rules are not defined for generated text, snippets, names, or denial-safe counts.
Fix: Define payload tiers and default notifications to opaque references until authorization is proven.
high

Audit availability is not a side-effect invariant

EXPERIENCE.md:156
Posted/approved success could be reported while evidence is missing or uncorrelated.
Fix: Require every side-effecting success state to show durable audit/correlation evidence or incident state.
high

API/admin parity is not a build contract

EXPERIENCE.md:271
The UX lacks operation-by-operation parity for states, denials, errors, and audit references.
Fix: Add a parity matrix for each admin action and API/client operation.

Accessibility review

review-accessibility.md

Solid accessibility floor, but high-risk workflows need deterministic focus, selection, live-region, and responsive-denial contracts.

high

Proposal version selection and focus recovery are under-specified

EXPERIENCE.md:171, EXPERIENCE.md:183
Approval posts the selected version, but version selection and focus fallback after row removal are undefined.
Fix: Specify version-list semantics, deterministic focus order, labelled editor, exact-version approval confirmation, and fallback focus targets.
medium

Live-region behavior is too broad

EXPERIENCE.md:184
Transitions are listed without politeness, atomicity, de-duplication, or source-specific announcement text.
Fix: Add a transition matrix for important state changes.
medium

Assistive localization is not explicit

DESIGN.md:139
Accessible names, descriptions, tooltips, and live-region text are not explicitly in the whole-string localization contract.
Fix: Extend FC-L10N requirements to non-visible assistive text.
