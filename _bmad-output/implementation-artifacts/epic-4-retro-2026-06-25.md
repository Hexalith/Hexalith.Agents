## 2. Deep Story Analysis — Synthesized Patterns

### What went well

**The deferred-fail-closed architecture paid off exactly as designed.** Epic 2's retro predicted "the bill for deferred everything comes due in Epic 4." Epic 4 absorbed the entire integration surface — but instead of the integration risk exploding, the team **published the full public contract surface and proved conformance over the deferred seams**, leaving live binding as cleanly-scoped future work. Every new surface (client facade, API routes, status/audit UI, launch-readiness) renders `PermissionDenied`/`Unavailable`/`Blocked` against the default DI graph, and that fail-closed default is the *tested* state. `Projections/` and `Application/Workflows/` remain `.gitkeep`-only by design.

**Reuse over reinvention was the explicit, enforced rule.** Each story built on shipped surface rather than re-deriving it: 4.3 reused 4.1's canonical enums + 4.2's query handlers + the 1.8/2.6/3.2/3.7 badges and presentation mappers; 4.4 mirrored Story 1.7's record-and-gate almost verbatim; 4.5 referenced existing per-story suites instead of re-authoring them. "Build on these, do NOT reinvent" reuse maps appear in every story's Dev Notes.

**Additive-first contract discipline (AD-17) held without exception.** Every contract change appended — enums carry `Unknown = 0` and serialize by name; new record fields land after the last positional parameter (`AgentStatusView.LaunchReadinessBlockers`/`ProductionLikeGenerationEnabled` after `ActivationBlockers`; `AgentInteractionGateCheck.LaunchReadiness` at ordinal 10). Each addition shipped round-trip + ordinal + no-leak tests.

**Governance was made explicit and machine-checked — the epic's standout asset.** Story 4.5's `traceability-manifest.json` + `TraceabilityManifestTests` assert (a) every FR-1..28 / NFR-1..10 / AD-1..19 / UX-DR1..41 id appears exactly once, (b) every named verification-path file exists on disk, and (c) every unresolved-governance id is flagged `blocker:true` and named in the report's blocker section. Unresolved decisions are surfaced as **explicit, enumerated launch blockers**, never hidden assumptions. Content-bearing audit stays blocked; production-like enablement fails closed by default precisely because audit-retention governance is unresolved.

**Sensitive-content confinement (AD-14) extended cleanly to the read/UI surfaces.** The new audit and operational surfaces are metadata-only: counts/rates are dimensioned solely by safe enums/ids/timestamps; no prompt/generated/edited/context content reaches any view, badge, `aria-label`, tooltip, `data-testid`, or telemetry dimension — verified by an auto-scanning contract test plus per-surface no-leak assertions.

### What was hard / where we struggled

- **Toolchain friction recurred, unchanged from Epic 2.** Solution-level `dotnet restore` exits 1 with no diagnostics; VSTest cannot open its TCP listener in the sandbox (`SocketException (13)`), forcing the team to run the built xUnit v3 executables directly, with `-m:1` serialized builds and `DiffEngine_Disabled=true`. Re-encountered and re-solved on essentially every story; never permanently fixed.
- **AC4 read-source reconciliation — a real tension, handled honestly.** Stories 4.3/4.4 require "recent Agent Call outcomes / terminal-state rates / posting outcomes," but there is **no live aggregate projection** (`Projections/` is `.gitkeep`-only). Rather than fabricate counts, the team shipped the safe *queryable summary surface* + query + result + fail-closed gateway and rendered an explicit "not available yet" affordance — repeating the Story 1.8 UX-DR2-vs-AC2 pattern. Correct, but it means a headline operational signal has a contract and a UI but no live data source yet.
- **A small spec/enum divergence, consciously accepted (4.3 review, LOW).** AC1 lists "content safety failures" and "generation failures" as *distinct* operational states, but the shipped canonical `AgentCallOperationStatus` folds content-safety into `GenerationFailed` (both → `RetryGeneration`). Left as-is to honor the "reuse the shipped enums 1:1 / no new aggregate-status members" mandate; provider failures stay distinct via `ProviderModelReadinessStatus.Failed`. Worth recording as a known, intentional doc-vs-enum gap.

### The genuinely valuable catches in review

Epic 4 had **0 Critical / 0 High** correctness defects, but the 4.5 adversarial review caught two MEDIUMs that matter:

- **A hollow verification path (MEDIUM, fixed).** The UI floor page-coverage meta-test enforced only nav-reachability, yet the report/manifest mapped **UX-DR15** (FcPageLayout composition) to it — claiming coverage no assertion actually provided. A future page could be nav-listed and silently skip the shell layout/state floor.
